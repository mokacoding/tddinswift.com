## Replace the Tailwind play CDN with a real build pipeline

Tailwind is currently loaded at runtime from `cdn.tailwindcss.com?plugins=typography`,
configured inline in `themes/blank-canvas/layouts/partials/head.html`.
This was a deliberate choice for simplicity but has known production downsides:

- FOUC on first paint (CSS is generated by JS in the browser).
- Runtime dependency on a third-party CDN; if it goes down or is blocked, the site is unstyled.
- Larger payload than a built+purged stylesheet (the full Tailwind engine ships to every visitor).
- Tailwind v3 only — the play CDN doesn't support v4.

Action: when the site goes "real" (custom domain launch, traffic that matters, or a perf pass),
wire up a proper build. Two reasonable options:

1. Hugo's `css.TailwindCSS` pipeline (requires a fresh `tailwind.config.js` with the correct content paths pointing at `themes/blank-canvas/layouts/**`, and a `main.css` with `@tailwind` directives).
2. A standalone Tailwind CLI build step driven from `package.json` scripts.

Either way, port the inline `tailwind.config = {...}` from `head.html` into the chosen config format,
swap the `<script src="cdn.tailwindcss.com">` for a `<link rel="stylesheet">` to the built file,
and verify visual parity against the current dev render before merging.

## Experiment: Inter for body text

The site currently uses the old WordPress pairing: Playfair Display for headings, Fira Sans for body — Fira being the original from the WordPress "Blank Canvas" theme.

Inter (by Rasmus Andersson) is the "boring default" of modern product UIs: wider x-height, tighter default spacing, more neutral than Fira.
Pairing Playfair + Inter gives one voice (Playfair) and a quiet frame around it, versus Playfair + Fira which gives two mildly-opinionated voices.

Try swapping body font from Fira Sans to Inter in the Tailwind font config, reload the landing page and a long post, and decide whether the quieter pairing reads better.
Flipping back is one line if it doesn't.
